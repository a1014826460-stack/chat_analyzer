# Context

## Glossary

- **线路 / 站点**
  指一个可独立获取开奖信息的外部开奖源，例如 `pc28`、`macao`、`australia`、`norway`。

- **线路选择**
  指选择开奖站点，以及与该站点绑定的期号、倒计时和开奖节奏解释规则；它不负责选择聊天数据库来源。

- **开奖数据**
  指某个线路返回的当前期号、开奖时间、下一期号、下一期开奖倒计时等信息。

- **账号**
  指聊天客户端中的一个用户身份。系统会用它去解析本地聊天配置目录，并定位对应的聊天数据库。

- **数据源**
  指用于读取聊天消息的本地文件来源，通常是聊天 SQLite 数据库，也可以是导出的文本文件。

- **账号与数据源**
  指先根据账号自动定位本地聊天数据库；如果自动定位失败，再允许手动指定聊天数据文件。

- **筛选条件**
  指作用于聊天消息读取和下注统计的全部条件，包括时间范围、群组范围、期号等条件。

- **期号筛选**
  指筛选条件中的期号条件。它默认跟随当前线路的下一期，也可以手动指定某一期号。

- **群组**
  指聊天记录中的一个会话分组，也是下注统计和群组屏蔽名单配置的基本作用域。

- **可见群组**
  指仅用于控制右侧图表和消息明细展示范围的群组视图条件。它不定义底层统计作用域。

- **下注记录**
  指从聊天消息中解析出来的一条投注事件。它可能来自直接下注消息，也可能来自回执型消息。

- **下注记录图表**
  指右侧用于展示当期下注变化的纵向堆积柱形图。它只统计 `大单`、`小单`、`大双`、`小双`、`大`、`小`、`单`、`双` 八类玩法；同一期内只追加增量层，只有期号变化才清空重开。

- **实时统计文本**
  指右侧用于追踪单笔下注的逐条明细视图。它与下注记录图表使用同一套八类玩法过滤规则，格式为 `时间 - 群聊名 - nickname - 下注种类 - 金额`。

- **统计结果**
  指按玩法汇总后的下注金额，以及用于图表和明细显示的可视化下注行。

- **汇总 totals**
  指按当前筛选作用域聚合后的玩法金额汇总。

- **屏蔽名单**
  指需要从统计中排除的人名规则集合。按作用域分为全局屏蔽名单和群组屏蔽名单。

- **群组屏蔽名单**
  指只对单个群组生效的排除人名集合。

- **全局屏蔽名单**
  指对所有群组统一生效的排除人名集合，适用于机器人等跨群固定账号。
