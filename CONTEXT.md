# Context

## Glossary

- **线路 / 站点**
  指一个可独立获取开奖信息的外部开奖源，例如 `pc28`、`macao`、`australia`、`norway`。

- **线路选择**
  指选择开奖站点，以及与该站点绑定的期号、倒计时和开奖节奏解释规则；它不负责选择聊天数据库来源。

- **开奖数据**
  指某个线路返回的当前期号、开奖时间、下一期号、下一期开奖倒计时等信息。

- **账号**
  指聊天客户端中的一个用户身份。系统会用它去解析本地聊天配置目录，并定位对应的聊天数据库。

- **数据源**
  指用于读取聊天消息的本地文件来源，通常是聊天 SQLite 数据库，也可以是导出的文本文件。

- **账号与数据源**
  指先根据账号自动定位本地聊天数据库；如果自动定位失败，再允许手动指定聊天数据文件。

- **筛选条件**
  指作用于聊天消息读取和下注统计的全部条件，包括时间范围、群组范围、期号等条件。

- **期号筛选**
  指筛选条件中的期号条件。它默认跟随当前线路的下一期，也可以手动指定某一期号。

- **群组**
  指聊天记录中的一个会话分组，也是下注统计和群组屏蔽名单配置的基本作用域。群组在业务上可同时表现为可读群名与稳定群 ID；凡涉及群类型记忆等持久化归类时，应以群 ID 作为主标识。

- **可见群组**
  指仅用于控制右侧图表和消息明细展示范围的群组视图条件。它不定义底层统计作用域。

- **下注记录**
  指从聊天消息中解析出来的一条投注事件。它可能来自直接下注消息，也可能来自回执型消息。

- **直接群**
  指以普通用户原始下注消息为主统计来源的群组类型。它的下注语义是增量事件流；主统计口径只读取用户下注信息，不依赖机器人回执信息。在期号识别上通常不依赖消息中显式携带期号，而是优先借助线路时间窗口、下注归期边界消息或最近一次可用期号上下文辅助判定归属期。

- **回执群**
  指以机器人生成的确认、回执或汇总消息为主统计来源的群组类型。它的下注语义是当期汇总快照；主统计口径直接使用机器人回执信息，不再回退为读取用户原始下注增量。消息通常可显式携带期号，并据此直接绑定统计期。

- **群类型**
  指某个群组在下注统计上的来源语义分类。当前仅区分为直接群与回执群两类。群类型允许在业务上发生切换，但系统应显式记忆每个群组的当前群类型，避免每次解析时临时摇摆并导致统计口径前后不一致。群类型一旦被确认为回执群，不应因短时间内未观察到回执消息而自动降级回直接群。系统可以基于消息特征发现某个直接群疑似已演变为回执群并给出提示，但最终是否切换群类型必须由人工确认。当前建议提示阈值为：在同一群组、10 分钟时间窗口内，连续出现至少 3 条符合标准回执格式的机器人消息，才触发“建议切换为回执群”的提示。群类型的持久化记忆应以群 ID 为主标识，而不是以可变的群名为主标识。

- **下注记录图表**
  指右侧用于展示当期下注变化的纵向堆积柱形图。它只统计 `大单`、`小单`、`大双`、`小双`、`大`、`小`、`单`、`双` 八类玩法；同一期内只追加增量层，只有期号变化才清空重开。

- **实时统计文本**
  指右侧用于追踪单笔下注的逐条明细视图。它与下注记录图表使用同一套八类玩法过滤规则，格式为 `时间 - 群聊名 - nickname - 下注种类 - 金额`。

- **统计结果**
  指按玩法汇总后的下注金额，以及用于图表和明细显示的可视化下注行。

- **汇总 totals**
  指按当前筛选作用域聚合后的玩法金额汇总。

- **屏蔽名单**
  指需要从统计中排除的人名规则集合。按作用域分为全局屏蔽名单和群组屏蔽名单。

- **群组屏蔽名单**
  指只对单个群组生效的排除人名集合。

- **全局屏蔽名单**
  指对所有群组统一生效的排除人名集合，适用于机器人等跨群固定账号。
- **机器人身份判定来源**
  指用于判定某个群组内哪个账号应被视为机器人账号的规则集合。当前规则约定为：优先使用 `nickname` 含“机器”的账号；如果该群在当前作用域内不存在这类账号，则回退为“最近 20 分钟内该群发言次数最多且消息格式符合机器人特征的账号”，并将判定结果按群组持久化，避免重复识别。
- **发送者角色**
  指一条聊天消息的发送方在业务上的身份类别。当前仅区分为机器人账号、普通用户账号与未判定账号三类。
- **消息语义类别**
  指一条聊天消息在业务流程中的功能类别。当前讨论范围内包括下注归期边界消息、机器人回执/汇总消息、开奖结果播报消息、机器人状态快照消息、用户下注消息、撤单/改注消息与普通聊天消息。
- **下注归期边界消息**
  指由机器人账号发出、用于划定某一期下注开始或结束边界的消息。当前约定为：开始标志通常包含“开始下注”或“下注开始”，结束标志通常包含“截止线”“封盘线”或“已截止”；同一期理论上只应出现一次开始标志与一次结束标志。若同一期出现后续同类重复边界消息，则仅第一条算有效边界，后续重复消息视为异常噪声，只用于原始聊天记录语义着色提示，不参与归期切分。回执群中的边界消息通常可直接携带期号；直接群中的边界消息允许不显式携带期号，此时需借助线路时间窗口或最近一次可用期号上下文辅助判定归属期。对于这类通过上下文补足归属期的边界消息，应在人工核对场景中与显式携带期号的边界消息区分看待。
- **机器人回执/汇总消息**
  指由机器人账号发出、用于表达某个用户当前期下注汇总状态的消息类别。当前最小识别标准为消息同时包含 `下注期数` 与 `下注内容` 两个核心字段。
- **开奖结果播报消息**
  指由机器人账号发出、用于播报某一期开奖结果或历史开奖序列的消息。它表达的是开奖事实，不直接承担下注归期切分，也不等同于用户下注回执或当期下注汇总，也不应作为群类型切换提示的证据来源。
- **机器人状态快照消息**
  指由机器人账号发出、用于展示某个用户当前积分、当期下注、冻结积分、盈亏或流水等账户状态的消息。它表达的是某一时点的账户与下注摘要，既不同于开奖结果播报消息，也不同于标准下注回执。即使其中包含“本期下注”字段，其语义仍应优先归为状态快照而非标准回执。
- **群类型切换**
  指某个群组在直接群与回执群之间发生来源语义变更后的重新归类过程。人工确认群类型切换后，应对该群当前统计作用域内的历史消息按新群类型口径重新解释，以维持同一群组统计结果的口径一致性。
- **机器人汇总对账**
  指将软件解析出的下注统计结果与机器人当期汇总消息进行逐项比较的校验过程。当前约定为按期号、按群组、按下注种类分别比较；若某下注种类的软件统计金额与机器人汇总金额的相对误差不超过 10%，则视为可接受。在人工核对展示中，这类处于容差范围内的结果应被明确标记为“通过（容差内）”，而不只是显示差值。
- **原始聊天记录语义着色**
  指在“原始聊天记录”视图中，基于发送者角色与消息语义类别进行差异化展示的方式。它服务于人工核对，不改变下注统计口径。对于通过上下文补足期号归属的下注归期边界消息，应与显式携带期号的边界消息使用不同视觉标识。该视图还应支持按语义类别筛选消息，而不只是依靠颜色区分。

- **群类型切换备注**
  指人工确认某个群组发生群类型切换时附带保存的说明信息，用于记录切换原因、判断依据或特殊背景，服务于后续审计与回溯。该备注默认应被提示填写，但允许留空并使用系统提供的默认说明模板。

- **群类型切换时间**
  指某个群组发生群类型切换并被人工确认时所记录的生效时间点，用于解释某一时间段内统计口径为何发生变化，并支持后续回溯。
